This option extracts either the Chameleon, Clover or Ozmosis bootlog from the IORegistry and saves it to a file named XXXXX_BootLog.txt, where XXXXX is the name, and possibly revision, of the bootloader. This file is saved in the latest dump/BootLog_Firmware folder.

The logs are dumped using the bdmesg tool by Kabyl, since modified by JrCs from an idea by STLVNUB.

Source Sourceforge - CloverPackage

If booting with OpenCore and there is a log available in either the Data hub or in a UEFI variable then this scan will extract it.

If OpenCore logging is set to write to file then you can get that by running the 'Bootloader Detect & Boot Sectors' scan.

Note OpenCore logging will have to have been enabled by the user. Consult the Configuration PDF for details.
